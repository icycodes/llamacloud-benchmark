# Update a Document in a LlamaCloud Managed Index Using `update_ref_doc` (Python) ## Background A real-world LlamaCloud RAG pipeline must support correcting wrong content in an existing index without rebuilding the whole pipeline. The managed `LlamaCloudIndex` exposes an `update_ref_doc(document)` method that upserts a document with an explicit `id_` (a `ref_doc_id`): if a document with that `id_` is already present, the managed pipeline replaces its content with the new content and reflects the change in subsequent retrievals. In this task you will build a Python program that: 1. Creates a brand-new LlamaCloud managed index from a small local corpus where every seed file is loaded as a `Document` with an explicit `id_` (the file's basename without extension). 2. After the index is fully ingested, replaces the content of one specific document (the one whose `id_` is `atlantis-aurora`) with a corrected version using `update_ref_doc(...)`. The corrected text is provided in a separate local file. 3. Performs retrievals before and after the update so that a downstream verifier can confirm that the new content is now searchable and that the old content is no longer surfaced.

# Extract PDF Tables to CSV Files Using LlamaParse Items Tree (Python)

## Background
LlamaCloud's LlamaParse service can return a structured **items tree** describing every typed element found on each page (tables, headings, figures, lists, etc.). For analytical pipelines you often want to pull every table out of a PDF and persist each one as its own CSV file so they can be loaded into pandas/Excel for downstream processing. In this task you must use the official `llama-cloud` Python SDK (the v2 API exposed via `from llama_cloud import LlamaCloud`) to parse a seeded financial PDF and write **one CSV file per table** found by LlamaParse.

The task workspace `/home/user/tables_task` is pre-seeded with a deterministic two-page PDF at `/home/user/tables_task/sample.pdf` that was generated when the environment was built. The PDF contains two distinct tables (one on each page):
- A `Quarterly Department Revenue` table with the columns `Department`, `Q1 Revenue`, and `Q2 Revenue`.
- A `Top Selling Products` table with the columns `Product` and `Units Sold`.

## Requirements
- Read the current `trial_id` from `/logs/artifacts/trial_id`.
- Write a Python script `extract_tables.py` in `/home/user/tables_task` that:
  - Authenticates against LlamaCloud using the `LLAMA_CLOUD_API_KEY` environment variable (read by the SDK from the environment).
  - Uses the official LlamaCloud Python SDK (`llama-cloud` package, imported as `from llama_cloud import LlamaCloud`) to upload `sample.pdf` with `purpose="parse"` and run a parse job with `tier="agentic"`, `version="latest"`, and `expand=["items"]` so that the structured items tree is returned.
  - Iterates over every page in `result.items.pages` and, for every item whose `type == "table"`, writes the table to its own CSV file at `/home/user/tables_task/tables/table_{n:03d}.csv` (where `n` starts at `1` and is incremented in the order tables are encountered while iterating pages in ascending `page_number` order and items in the natural list order). The CSV file must be UTF-8 text and must contain every row of the table (header row included), with one CSV row per table row.
  - Writes a log file `/home/user/tables_task/output.log` summarising the run (see Acceptance Criteria for the exact lines that must appear).
- The script must run end-to-end with `python3 extract_tables.py` from `/home/user/tables_task` and exit with status code `0`.

## Implementation Hints
- Install package: `llama-cloud>=2.1` is already pre-installed in the environment.
- Instantiate `LlamaCloud()` with no arguments to pick up `LLAMA_CLOUD_API_KEY` automatically.
- Upload the file with `client.files.create(file="./sample.pdf", purpose="parse")` and pass `file_id=file.id` into `client.parsing.parse(...)`. The SDK polls for completion internally.
- The structured items tree is returned when you request `expand=["items"]`. Each table item exposes an `rows` attribute (a list-of-lists) and a pre-formatted `csv` attribute (a CSV string for the same table). Either can be used to write the CSV file — the simplest path is to write the `csv` field directly if it is non-empty; otherwise fall back to building the CSV from `rows` with Python's `csv` module.
- The output CSV files live under `/home/user/tables_task/tables/`. Create the directory if it does not already exist.
- Read `trial_id` once at the top of the script and embed it in `output.log` so the verifier can correlate the run with the trial.

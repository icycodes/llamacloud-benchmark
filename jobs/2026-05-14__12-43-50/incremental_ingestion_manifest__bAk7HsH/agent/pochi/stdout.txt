# Incremental Ingestion Plan for a LlamaCloud Index ## Background LlamaCloud is LlamaIndex's managed RAG platform. The `client.pipelines.files.create()` endpoint uses **additive/upsert semantics** — files NOT included in the request remain unchanged in the index, and files INCLUDED are added or re-indexed. To keep a managed index in sync with a local folder *cheaply*, you should only upsert files whose content has actually changed and only delete files that disappeared locally. In this task you will build the planning half of that workflow: a script that computes a content-addressed manifest of a local folder, diffs it against a prior manifest, and writes a structured `upload_plan.json` describing exactly which files to upload, update, delete, or skip. The script must also construct a `llama_cloud.LlamaCloud` client (sanity check that the SDK and API key are wired up correctly) but it MUST NOT make any network calls. ## Pre-existing Layout The project directory `/home/user/incremental_ingest` already contains: ``` /home/user/incremental_ingest/ ├── data/ │ ├── intro.txt # content unchanged since last sync │ ├── glossary.txt # content has been edited since last sync │ └── changelog.txt # brand-new file, not in the prior manifest └── prior_manifest.json # snapshot of the previous sync, contains entries for intro.txt, glossary.txt, and removed_doc.txt ``` The schema of `prior_manifest.json` is: ```json { "files": { "<relative_path>": {"sha256": "<hex sha256>", "size": <int bytes>} } } ``` `removed_doc.txt` exists ONLY in `prior_manifest.json` — there is NO physical file for it on disk anymore. Your script must treat that as a deletion. ## Requirements Create the following inside `/home/user/incremental_ingest`: ### 1. `sync.py` A standalone script that, when invoked as `python3 sync.py`, performs the following steps in order: 1. **Validate environment**: Read `LLAMA_CLOUD_API_KEY` from `os.environ`.

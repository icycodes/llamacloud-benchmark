# Extract a Resume to Structured JSON with LlamaExtract and a Pydantic Schema ## Background LlamaCloud is LlamaIndex's managed RAG-as-a-Service platform. In addition to LlamaParse (which converts visually rich documents to clean markdown), LlamaCloud also exposes **LlamaExtract**, a structured-data extraction API. Instead of returning markdown for you to post-process, LlamaExtract takes a **data schema** (defined as a Pydantic model or raw JSON Schema) and returns a structured object that conforms to it. The canonical pattern with the `llama-cloud` Python SDK is: 1. Define a `pydantic.BaseModel` describing the fields you want. 2. Upload the source document with `client.files.create(file=..., purpose="extract")`. 3. Submit an extraction job with `client.extract.create(file_input=..., configuration={"data_schema": Model.model_json_schema(), "extraction_target": "per_doc", "tier": "agentic"})`. 4. Poll `client.extract.get(job.id)` until `status` is `"COMPLETED"` (or `"FAILED"`/`"CANCELLED"`). 5. Read `job.extract_result` (a dict matching your schema) and persist it as JSON. In this task you will build that pipeline end-to-end against a single resume document and persist the extracted structured output to disk. ## Pre-existing Layout The project directory `/home/user/resume_extract` already contains exactly one file, `resume.txt`, with this content: ``` Candidate Name: Jane Doe Email: jane.doe@example.com Phone: 555-123-4567 Technical Skills: Python, SQL, AWS, Docker, Kubernetes Experience: - Senior Software Engineer at AcmeCorp (2020-Present) - Software Engineer at Beta Inc (2018-2020) Education: BS Computer Science, MIT, 2018 ``` ## Requirements Create a Python script `/home/user/resume_extract/extract_resume.py` that: 1. **Defines a Pydantic schema** named `Resume` at module top level. It MUST be a subclass of `pydantic.BaseModel` and contain exactly these three fields (use `pydantic.Field` with `description=...` so the LLM gets context): - `name: str` — Full name of the candidate. - `email: str` — Email address of the candidate. - `skills: list[str]` — Technical skills and technologies mentioned in the resume. 2. **Reads the API key** from the `LLAMA_CLOUD_API_KEY` environment variable. Do NOT hardcode the key. If the variable is unset or empty, exit with a non-zero status code and print an error to stderr. 3. **Instantiates a client** via `from llama_cloud import LlamaCloud` and either `LlamaCloud()` (the SDK reads `LLAMA_CLOUD_API_KEY` from the environment automatically) or `LlamaCloud(api_key=os.environ["LLAMA_CLOUD_API_KEY"])`. 4. **Uploads `resume.txt`** with `client.files.create(file="./resume.txt", purpose="extract")` and captures the returned `file_obj.id`. 5. **Submits an extract job** with `client.extract.create(...)`: - `file_input=<uploaded file id>` - `configuration={"data_schema": Resume.model_json_schema(), "extraction_target": "per_doc", "tier": "agentic"}` 6. **Polls** `client.extract.get(job.id)` in a loop, sleeping 2 seconds between iterations, until the job status is one of `"COMPLETED"`, `"FAILED"`, or `"CANCELLED"`. Create `extract_resume.py`. Reference implementation: ```python import json import os import sys import time from pydantic import BaseModel, Field from llama_cloud import LlamaCloud class Resume(BaseModel): name: str = Field(description="Full name of the candidate") email: str = Field(description="Email address of the candidate") skills: list[str] = Field( description="Technical skills and technologies mentioned in the resume" ) def main() -> int: api_key = os.environ.get("LLAMA_CLOUD_API_KEY", "").strip() if not api_key: print("LLAMA_CLOUD_API_KEY is not set", file=sys.stderr) return 2 client = LlamaCloud() # reads LLAMA_CLOUD_API_KEY from the environment file_obj = client.files.create(file="./resume.txt", purpose="extract") job = client.extract.create( file_input=file_obj.id, configuration={ "data_schema": Resume.model_json_schema(), "extraction_target": "per_doc", "tier": "agentic", }, ) while job.status not in ("COMPLETED", "FAILED", "CANCELLED"): time.sleep(2) job = client.extract.get(job.id) if job.status != "COMPLETED": print(f"extract job ended with status: {job.status}", file=sys.stderr) return 3 with open("/home/user/resume_extract/extracted.json", "w", encoding="utf-8") as f: json.dump(job.extract_result, f, indent=2) return 0 sys.exit(main()) ``` 3.
